管理、监控和部署edit

本书的主要目标是使用 Elasticsearch 作为后端创建应用程序。本章稍微有点不同。在这里，你将学习到如何管理 Elasticsearch 自身。Elasticsearch 是一个复杂的软件，有许多可移动组件。大量的 API 被设计用来帮助管理你的 Elasticsearch 部署。

在本章中，将涵盖三个主要的话题：

  • 监视集群的重要的统计数据，了解哪些行为是正常的，哪些应该引起警惕，并解释Elasticearch提供的各种统计数据。
  • 部署集群到生产环境，包括最佳实践和应该（或不应该！）修改的重要配置。
  • 部署后的维护，比如 滚动重启(rolling restart) 或 集群的备份
