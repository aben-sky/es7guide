监控edit

Elasticsearch 经常以多节点集群的方式部署。有多种 API 让你可以管理和监控集群本身，而不用和集群里存储的数据打交道。

和 Elasticsearch 里的大多数功能一样，有一个总体的设计目标，即：任务应该通过 API 执行，而不是通过修改静态的配置文件。这一点在你的集群扩容时尤为重要。即便使用配置管理系统（比如 Puppet，Chef，Ansible），一个 HTTP API 调用，也比将新的配置文件往上百台物理设备上推送简单得多。

因此，本章将介绍允许动态调整、调优和配置集群的 各种API。同时，还会介绍一大堆提供集群自身统计数据的 API，可以用来监控集群健康状态和性能。
