Marvel 监控edit

Marvel 让你可以很简单的通过 Kibana 监控 Elasticsearch。你可以实时查看你的集群健康状态和性能，也可以分析过去的集群、索引和节点指标。

虽然你可以通过本章介绍的 API 查看大量的统计数据，但它们只能显示在单个时间点发生的情况。了解这个瞬间的内存占用比当然很有用，但是了解内存占用 随时间的趋势 更加有用。Marvel 会查询并聚合这些数据，你可以通过可视化效果看到集群随时间的变化，这样就能很容易的看到趋势。

随着集群规模的增长，统计 API 的输出内容会复杂得让人完全没法看。当你有十几个节点，更别说一百个了，阅读一堆 JSON 就非常乏味了。而 Marvel 可以让你以交互式的方式浏览数据，并使你可以更轻松地了解特定节点或索引正在发生着什么事情。

Marvel使用与您可以使用的相同的统计API​，它不公开任何您无法通过API访问的统计信息。然而，Marvel大大简化了这些统计数据的收集和可视化工作。

Marvel 可以免费使用（包括生产环境！），所以你一定要试试！安装介绍参阅 Marvel 入门。
