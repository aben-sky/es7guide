文件描述符检查 (File descriptor check)

文件描述符是用于跟踪打开的"文件"的 Unix 构造。 但在 Unix 中， 一切都是一个文件。 例如，"文件"可以是物理文件、虚拟文件（例如 /proc/loadavg）或 网络套接字。 Elasticsearch 需要大量的文件描述符(例如，每个分片由多个段和其他文件组成，以及到其他节点的连接，等等)。 这个引导检查在 OS X 和 Linux 上是强制的。 要通过文件描述符检查，你可能需要配置文件描述符。
