最大文件大小检查 (Max file size check)

作为单个分片组件的 段文件(segment files) 和作为事务日志组件的 事务日志(translog) 生成可能会变得很大(超过数GB)。 在 Elasticsearch 进程可以创建的最大文件大小有限的系统上，这可能会导致写入失败。 因此，此处最安全的选项是最大文件大小不受限制，而这正是最大文件大小引导检查强制执行的。 要通过最大文件检查，必须配置系统，以允许 Elasticsearch 进程能够写入无限大的文件。 这可以通过在配置文件 /etc/security/limits.conf 中将 fsize 设置为 unlimited 来实现（注意，可能还需要增加 root 用户的限制）。
