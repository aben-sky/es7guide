Elasticsearch是什么?edit

你知道, 为了搜索(和分析)

Elasticsearch 是分布式的搜索和分析引擎，是 Elastic Stack 的核心组件。 Logstash 和 Beats 用于收集、聚合和丰富数据，并将其存储在Elasticsearch中 Kibana让你能够交互式地探索、可视化、分享对数据的理解，并管理和监控 Elastic Stack。 Elasticsearch 是索引、搜索和分析魔法发生的地方。

ElasticSearch 为所有类型的数据提供了接近实时的搜索和分析。 无论你是否有结构化或非结构化的文本、数值或地理空间数据，Elasticearch 都可以在支持快速搜索的方式下有效地存储和索引。 比简单的数据检索和聚合信息更有意义的，是发现数据中的趋势和模式。 随着数据和查询量的增长，Elasticsearch 的分布式特性使你能无缝的部署更多节点。

虽然并不是每个问题都是一个搜索问题，但 Elasticsearch 提供了在多种用例中处理数据的速度和灵活性：

  • 向应用程序或网站添加搜索框
  • 存储和分析日志、度量和安全事件数据
  • 使用机器学习自动对数据的行为进行实时建模
  • 使用 Elasticsearch 作为存储引擎实现业务工作流的自动化
  • 使用 Elasticsearch 作为地理信息系统(geographic information system, GIS)管理、集成和分析空间信息
  • 利用 Elasticsearch 作为生物信息学研究工具存储和处理遗传(genetic)数据

我们不断对人们使用搜索的新颖方式感到惊讶。 但是，无论是你的用例与其中一个相似，还是使用 Elasticearch 来解决一个新问题，在 Elasticearch 中处理数据、文档和索引的方式都是一样的。
