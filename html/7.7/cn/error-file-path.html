JVM 致命错误日志

默认情况下，Elasticsearch 配置 JVM 将致命错误日志写入默认日志目录(RPM 和 Debian 包发行版 下是 /var/log/elasticsearch，而 tar 和 zip 归档发行版 则是在 Elasticsearch 安装根目录下的 logs 目录)。 这些是 JVM 遇到致命错误(例如，段错误)时产生的日志。 如果此路径不适合接收日志，您应该修改 jvm.options 中的 -XX:ErrorFile=... 条目。
