重要的系统配置

理想情况下，Elasticsearch 应该在一台服务器上单独运行，并使用所有可用的资源。 为此，需要配置你的操作系统，允许运行 Elasticsearch 的用户访问比默认情况下允许的更多的资源。

在投入生产之前，必须 考虑以下设置：

  • 禁用交换 (Disable swapping)
  • 增加文件描述符 (Increase file descriptors)
  • 确保足够的虚拟内存 (Ensure sufficient virtual memory)
  • 确保足够的线程 (Ensure sufficient threads)
  • JVM DNS缓存设置 (JVM DNS cache settings)
  • 不使用noexec挂载临时目录 (Temporary directory not mounted with noexec)

开发模式 vs 生产模式

默认情况下，Elasticsearch 假设是在开发模式下工作。 如果上述任何设置配置不正确，将在日志文件中写入警告，但你仍然可以启动并运行 Elasticsearch 节点。

一旦配置了 network.host 这样的网络设置，Elasticsearch 就会认为你正在进入生产环境，并将上述警告升级为异常。 这些异常将阻止 Elasticsearch 节点启动。 这是一项重要的安全措施，可以确保不会因为服务器配置不当而丢失数据。
